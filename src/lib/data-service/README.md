# Cloud-DB

This module handles the access to data from our servers.
Its main export is a controller which is what we use to make all requests.

## Folder structure

- **cloud-db-controller.js**
  Controller with information about the user and session, and allows one to make requests to our cloud.

- **hadron-public-http-api.js**
  Gateway to the Hadron Api that can only be used for its public endpoints.
  Useful when we don't need the user to be logged.

- **auth/**
  Everything related to creating and maintaining a logged session with hadron using different providers.

  The controller `auth/session-controller.js`can be used if one needs to use the session/user _only_, but generally is better to use`cloud-db-controller.js`, which also includes the possibility of making request.

- **hadron-http-api/**
  The HTTP Api and it's types are auto generated at hadron-server (ATM updated here manually).
  Requests also need to be properly signed for security.

  - _api.types.d.ts_ - Autogenerated at hadron-server.
  - _hadron-http-api_ - Autogenerated at hadron-server. Gateway to the Hadron Api
  - _aws-sv4-fetch_ - Signing request to be able to reach AWS private endpoints
